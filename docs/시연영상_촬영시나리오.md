## 샘플 영상에서 가져온 4가지 뼈대

샘플(반도체 EAP 도메인 LATS 에이전트)의 득점 구조를 분석하면 이렇다. 우리도 이 4박자를 그대로 쓴다.

1. **제품 UI에서 실제 실행** — Chainlit 챗봇에 질문 → 답변. (우리: React 파이프라인 UI + 🧠/⚙️ 추론 타임라인)
2. **근거[출처]를 화면에 노출** — 답변마다 `[출처: EAP 소스코드, PLC MAP 사양서 …]`. (우리: 검증 사실·Job 근거·grounding 플래그)
3. **관측 대시보드로 전환해 내부 추론을 증명** — LangSmith 트레이스 트리(서브에이전트·토큰·비용·지연)를 확대. (우리: **Langfuse** `agentic-supervisor-graph` 트레이스 — 이번에 배선 완료)
4. **알고리즘/아키텍처 한 장** — LATS 플로우차트 슬라이드. (우리: Supervisor–Observation 재계획 루프 다이어그램 = 발표 덱 슬라이드3 재사용)

> 핵심 리듬: **"제품 화면에서 답이 나온다 → 뒤로 돌아가서 이렇게 멀티에이전트가 판단했다"** 를 2번 반복한다.

## 촬영·편집 강조 포인트 (샘플에서 배운 것)

- **탭 전환으로 리듬:** 제품(localhost) ↔ Langfuse(us.cloud.langfuse.com). 샘플이 Chainlit↔LangSmith를 오간 것과 동일.
- **트레이스 트리는 '펼친 상태'로:** Langfuse에서 Supervisor/서브에이전트 Generation 노드를 펼쳐 **토큰·지연·프롬프트**가 보이게. 샘플이 `impact_analyst`·`vector_search`·비용($0.0549)·28.4K 토큰을 노출한 것과 같은 효과.
- **🧠/⚙️ 배지 대비를 주인공으로:** UI 타임라인에서 `🧠 LLM`(판단)과 `⚙️ 규칙`(검증)이 번갈아 찍히는 걸 확대. "LLM은 판단, 코드는 확정"이 한눈에.
- **근거 필드에 박스/화살표:** 반려 메일·교정 제안의 `근거`, grounding 플래그를 강조. 샘플의 `[출처: …]`에 대응.
- **일부러 실패를 하나 보여줘라:** 2:05–2:55의 job_not_found→복구가 최대 득점. 성공만 나열하면 저수준으로 보인다(가이드 명시).
- **대기 컷/배속:** LLM 응답 대기(중앙 2.46초)·수집 로딩은 컷 또는 1.5~2배속. 과정(추론)에 시간을 쓰고 대기에는 쓰지 않는다.
- **5분 초과 금지**, 결과·핵심 로그를 앞쪽에 배치. 파일명 `08079_이형진_시연영상.mp4`.

## 우리 vs 샘플 — 오히려 유리한 차별점 (Q&A 대비)

- 샘플은 답변 정확성 위주. 우리는 **엑셀 검증을 결정론 코드로 분리**해 재현성 표준편차 0을 증명 → "왜 검증까지 LLM에 안 맡겼나"에 정량 답변 가능.

- 관측은 샘플=LangSmith, 우리=**Langfuse**로 동급. 둘 다 프롬프트·토큰·지연·비용을 트레이스로 증명.
