# 시연 영상 대본 (5분 이내)

> 제출 파일명: `08079_이형진_시연영상.mp4` · MP4 · 5분 이내 · 편집 권장
> 핵심 원칙(가이드): **결과보다 '과정(추론 로그)'이 중요**. 단순 Tool Call이 아니라
> **LLM이 판단(계획·제안)하고 코드가 검증**하는 에이전트 흐름을 확대해서 보여줄 것.
> 각 단계 판단은 **실행 트레이스 파일(`data/traces/{요청ID}.md`)로 자동 기록**되어 증거로 남는다.

## 사전 준비
- 터미널 글자 크기 키우기(녹화 시 가독성). 다크 테마 권장.
- 명령 1개로 전체 흐름이 출력됨:
  ```bash
  set PYTHONUTF8=1
  python backend/cli.py demo
  ```
- **UI 데모(권장)**: `cd frontend && npm run dev` → 02 검증 실행 후 **‘에이전트 추론 타임라인’ 패널**과
  **‘추론 트레이스(.md)’ 다운로드 버튼**을 화면에 보여준다. 각 스텝에 `🧠 LLM` / `⚙️ 규칙` 배지가 붙는다.
- 실행이 끝나면 `data/traces/{요청ID}.md` 가 생성된다 — 이 파일을 열어 **단계별 판단 주체 표**를 그대로 보여주면
  "과정을 기록·증명한다"는 요구를 충족한다.

## Gmail MCP 스타일 기반 요청 초안 (신규)

1. (Claude Code) 팀장이 포워딩한 취합 메일을 Gmail MCP로 검색·열람한다.
2. (Claude Code) 내 Sent에서 과거 취합요청 메일 N개를 가져와 `/api/save-style-mail`로 저장한다.
3. 포워딩 메일 제목/본문을 앱 입력에 붙여넣고 '가이드/메일 초안 생성'을 실행한다.
4. 초안이 '내 과거 발송 스타일 반영됨' 배지와 함께 생성된다.
5. 확인·수정 후 Claude Code가 `create_draft`로 Gmail 임시보관함에 초안을 저장한다.

핵심: 앱은 Gmail을 직접 읽지 않는다. 읽기/드래프트는 Claude Code(MCP)가 수행하고 앱은 텍스트만 처리한다.
