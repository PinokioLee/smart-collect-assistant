# Smart Collect 시연 영상 녹음 대본

## 시연 개요

- 목표 시간: 5분 이내
- 녹화 방식: 화면 조작과 발표자 음성 녹음
- 자막: 없음
- 녹화용 접속 주소: `http://127.0.0.1:5173/?presentation=1`
- 시연 환경: Gmail API 기반 기능을 Mock 메일로 재현
- 핵심 사례
  1. 정상 제출 및 최종 취합 완료
  2. 작업번호가 없는 제출 메일 복구

---

## 0:00~0:35 — 시연 환경 설명

### 화면 조작

1. Smart Collect의 초기 화면을 보여준다.
2. `전체` 필터가 선택되어 있고 메일 목록이 비어 있는 상태를 보여준다.

### 발표 대본

> 지금부터 사내 자료 취합 업무를 자동화하는 프로그램을 시연하겠습니다.
>
> 사내 메일은 보안상 사용할 수 없어 Gmail API를 이용해서 메일을 가져오도록 구현했습니다.
>
> 그리고 스케줄러가 백그라운드에서 실행되어 정해진 시간마다 새로운 메일을 자동으로 가져오도록 개발하였으나,
>
> 오늘은 짧은 시간 안에 해당 기능을 보여드려야 하기 때문에 수동으로 메일을 수집하는 방법으로 영상을 촬영하였습니다.

---

## 0:35~1:00 — 메일 수신 및 Agent 자동 처리

### 화면 조작

1. `지금 메일 확인` 버튼을 클릭한다.
2. 처리가 끝날 때까지 기다린다.
3. 상단 처리 현황과 메일 목록을 천천히 보여준다.
4. `전체 / 처리 완료 / 확인 필요 / 격리` 필터를 차례로 짧게 보여준다.

### 발표 대본

> 현재 화면은 아직 처리한 메일이 없는 초기 상태입니다.
>
> 지금 메일 확인 버튼을 누르면 Inbox Intake Agent가 메일의 제목과 본문, 발신자, 첨부파일을 분석하여 일반 문의, 자료 제출, 수정 제출, 위험 메일 등의 업무 유형으로 분류합니다.
>
> Supervisor Agent는 분류 결과와 위험도를 바탕으로 다음 처리 경로를 결정합니다. 자료 제출은 Validation Agent와 Validation/Merge Agent가 파일을 검증하고 병합하며, 업무 질문은 Q&A RAG/Communication Agent가 근거를 찾아 답변을 작성합니다.
>
> 판단이 불확실한 메일은 Supervisor Agent가 사람의 검토 대상으로 전달하고, 위험 메일은 Security Agent가 격리합니다.
>
> 처리 결과는 전체, 처리 완료, 확인 필요, 격리 상태로 나누어 확인할 수 있습니다.
>
> 시간 관계상 모든 사례를 시연할 수는 없어, 이 프로젝트의 핵심 기술이 적용된 두 가지 사례만 보여드리겠습니다.
>
> 먼저 정상 제출과 최종 취합 과정을 확인하고, 다음으로 작업번호가 없는 제출 메일을 복구하는 과정을 보여드리겠습니다.

---

# CASE 1. 정상 제출 및 최종 취합 완료

## 1:00~1:50 — 앱 처리 결과

### 화면 조작

1. `처리 완료` 필터를 클릭한다.
2. 정상 제출 또는 수정본 제출 완료 카드를 선택한다.
3. 메일의 첨부파일명과 판단 근거를 보여준다.
4. `전체` 필터로 전환한다.
5. 최종 취합 완료 카드를 찾아 최종 병합 파일명과 완료 메일 초안을 보여준다.

### 발표 대본

> 첫 번째는 정상적으로 제출된 자료가 자동 처리되는 과정입니다.
>
> Inbox Intake Agent가 이 메일을 자료 제출로 분류하면 Supervisor Agent가 Validation Agent로 전달합니다.
>
> Validation Agent는 메일에서 작업번호를 확인하고 첨부된 엑셀 파일을 해당 취합 업무와 연결합니다.
>
> 화면에는 실제로 수신한 첨부파일명이 표시됩니다.
>
> 제출 파일은 바로 병합하지 않고 필수 컬럼과 데이터 형식을 먼저 검증합니다. 검증에 통과한 데이터만 기존 취합 자료에 병합하고 정상 접수 결과를 기록합니다.
>
> 모든 담당자의 제출이 완료되면 Final Validation Agent가 최종 결과를 확인하고, Completion/Report Agent가 담당자에게 전달할 완료 메일 초안을 작성합니다.
>
> 시연 환경에서는 안전을 위해 실제 메일 자동 발송은 비활성화되어 있습니다.

## 1:50~2:10 — Langfuse 및 SQLite 확인

### 화면 조작

1. Langfuse에서 정상 제출 Trace를 연다.
2. `inbox_classification`과 `supervisor_route`를 차례로 보여준다.
3. 완료 메일을 생성한 Generation을 짧게 보여준다.
4. PowerShell의 SQLite 조회 화면으로 전환한다.
5. 제출 상태, 검증 결과, 최종 파일 기록을 보여준다.

### 발표 대본

> Langfuse에서는 메일 분류와 Supervisor의 라우팅, 파일 처리 이후 완료 메일 생성까지 Agent별 실행 과정을 확인할 수 있습니다.
>
> SQLite에는 제출 상태와 검증 결과, 최종 파일 정보가 저장되어 실제 업무 처리 결과의 일관성을 유지합니다.

# CASE 2. 작업번호 없는 제출 메일 복구

## 2:10~2:55 — 앱 처리 결과

### 화면 조작

1. 앱으로 돌아간다.
2. `확인 필요` 필터를 클릭한다.
3. `작업번호 없는 제출` 메일 카드를 찾는다.
4. 카드의 `판단 근거와 실행 로그`를 펼친다.
5. 내용을 보여준 뒤 `작업번호 없는 제출 복구` 버튼을 클릭한다.

### 발표 대본

> 두 번째는 제출 파일은 있지만 기존 취합 건을 식별할 작업번호가 없는 경우입니다.
>
> Inbox Intake Agent는 첨부파일과 본문을 분석해 자료 제출 메일로 분류합니다. 그러나 Validation Agent가 연결할 작업번호를 찾지 못하면 처리 결과를 Supervisor Agent에게 다시 전달합니다.
>
> Supervisor Agent는 잘못된 취합 건에 연결되는 것을 방지하기 위해 자동 처리를 중단하고 사람의 검토가 필요한 경로로 전환합니다.
>
> 실행 로그를 보면 제출 메일로 분류된 이후 작업번호를 찾지 못해 복구 경로로 이동한 것을 확인할 수 있습니다.
>
> 담당자가 올바른 취합 작업을 확인한 뒤 복구 버튼을 누르면 해당 제출 자료를 기존 작업과 안전하게 연결하고 중단된 지점부터 처리를 다시 이어갑니다.

## 2:55~3:20 — Langfuse 및 SQLite 확인

### 화면 조작

1. Langfuse에서 해당 메일의 Trace로 전환한다.
2. `inbox_classification`과 `supervisor_route` 결과를 보여준다.
3. PowerShell의 SQLite 조회 화면으로 전환한다.
4. 작업번호를 찾지 못한 기록과 복구 이후의 처리 기록을 보여준다.

### 발표 대본

> Langfuse에서는 제출 메일로 판단한 분류 결과와 Worker 실행 실패 이후 Supervisor가 선택한 안전한 처리 경로를 확인할 수 있습니다.
>
> SQLite에는 처음 작업번호를 찾지 못한 상태와 담당자가 복구한 결과가 함께 기록됩니다.
>
> 따라서 어떤 예외가 발생했고 어떻게 복구되었는지 사후 추적이 가능합니다.

---

## 3:20~3:40 — 마무리

### 화면 조작

1. 앱으로 돌아간다.
2. `전체` 필터를 클릭한다.
3. 상단 처리 현황과 전체 메일 목록을 보여준다.
4. 화면을 움직이지 않고 마지막 설명을 마친다.

### 발표 대본

> 지금까지 정상 제출부터 최종 취합까지의 자동 처리와 작업번호가 없는 예외 상황의 복구 과정을 확인했습니다.
>
> 이 시스템은 단순히 메일을 분류하는 데서 끝나지 않고, 여러 Agent의 판단과 실제 업무 상태를 연결하고 모든 처리 과정을 추적 가능하게 만든 것이 핵심입니다.

## 녹화 팁

- 화면을 클릭한 직후 말하지 말고, 결과가 나타난 다음 설명한다.
- 마우스를 빠르게 흔들지 말고 다음에 보여줄 위치로 천천히 이동한다.
- Langfuse에서는 입력 전체를 읽지 말고 `category`, `action`, `reason`만 보여준다.
- SQLite에서는 모든 행을 읽지 말고 이벤트 ID, 상태, Agent 행동 결과만 보여준다.
- 한 화면을 보여주는 동안 문장을 2~3개 이상 길게 이어가지 않는다.
- 실제 발송이 비활성화된 시연 환경이라는 점은 한 번만 설명한다.
- 녹화 전 알림, TV, 메신저 소리를 끄고 마이크 입력 레벨을 확인한다.
