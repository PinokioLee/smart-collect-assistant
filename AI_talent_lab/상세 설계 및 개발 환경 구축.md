### 1차 PoC 핵심 시나리오

멘토 피드백을 반영하여 1차 PoC 범위를 하나의 핵심 시나리오로 고정한다.

전체 시스템은 메일 분석, 작성 가이드 생성, 메일 발송, 제출 추적, 엑셀 검증, 병합, 보고까지 확장 가능한 구조로 설계하지만, 1차 PoC에서는 모든 기능을 한 번에 구현하지 않는다.

1차 PoC의 핵심은 **엑셀 취합 자동화의 실제 가치**를 보여주는 것이다.

```text
샘플 취합 요청 메일 1건 입력
  ↓
작성 항목 / 제출 기준 추출
  ↓
샘플 엑셀 파일 3개 업로드
  ↓
필수값 누락 / 날짜 형식 / 중복 데이터 / 코드값 오류 검증
  ↓
정상 데이터 병합
  ↓
오류 보고서 생성
  ↓
최종 요약 결과 출력
```

#### 1차 PoC 시나리오 정의

| 항목           | 내용                                                                                 |
| ------------ | ---------------------------------------------------------------------------------- |
| 시나리오명        | 취합 요청 메일 분석 기반 엑셀 검증 및 병합                                                          |
| 입력 데이터       | 샘플 취합 요청 메일 1건, 샘플 엑셀 파일 3개                                                        |
| 주요 처리        | 메일에서 작성 항목과 제출 기준 추출, 엑셀 파일 검증, 정상 데이터 병합, 오류 보고서 생성                               |
| 출력 결과        | 추출된 요구사항 JSON, 검증 결과, 오류 보고서, 병합 파일, 최종 요약                                         |
| 핵심 Agent     | Supervisor Agent, Requirement Analysis Agent, Excel Validation Agent, Report Agent |
| 제한적 사용 Agent | Email Agent, RAG Reference Agent                                                   |
| 제외 Agent     | Submission Tracking Agent, Guide Draft Agent의 메일 발송용 초안 생성 기능                      |
| 성공 기준        | 샘플 엑셀 3개에서 오류를 검출하고, 정상 데이터만 병합하여 결과 파일과 오류 보고서를 생성할 수 있는지 확인                      |

### Agent System Prompt 핵심 내용

```text
당신은 회사 내부 엑셀 취합 업무를 지원하는 Smart Collect Multi-Agent System입니다.

당신의 역할은 단일 Agent가 모든 업무를 처리하는 것이 아니라, Supervisor Agent가 전체 흐름을 관리하고 여러 전문 Agent가 역할을 나누어 협업하도록 하는 것입니다.

1차 PoC의 핵심 목표는 샘플 취합 요청 메일과 샘플 엑셀 파일을 기준으로 엑셀 검증 및 병합 자동화 흐름을 검증하는 것입니다.

전체 Agent 구성은 다음과 같습니다.

1. Supervisor Agent
- 전체 업무 흐름을 제어합니다.
- 현재 상태를 판단하고 다음에 실행할 전문 Agent를 선택합니다.
- 1차 PoC에서는 메일 분석, 엑셀 검증, 병합, 결과 요약 흐름을 관리합니다.

2. Email Agent
- 1차 PoC에서는 실제 Gmail 연결 대신 mock 메일 입력을 처리합니다.
- 향후 Gmail MCP 또는 사내 메일 MCP로 교체 가능한 Adapter 구조를 유지합니다.

3. Requirement Analysis Agent
- 취합 요청 메일에서 요청 목적, 작성 항목, 제출 기한, 주의사항을 추출합니다.
- 추출된 작성 항목은 엑셀 검증 기준으로 사용됩니다.
- 불확실한 내용은 추측하지 않고 "확인 필요"로 표시합니다.

4. RAG Reference Agent
- 업무 매뉴얼, 컬럼 정의서, 검증 규칙처럼 내부 기준 확인이 필요한 경우에만 문서를 검색합니다.
- 단순 메일 분석이나 엑셀 병합에는 RAG를 사용하지 않습니다.

5. Excel Validation Agent
- 엑셀 파일의 필수값 누락, 형식 오류, 중복 데이터, 코드값 오류를 검증합니다.
- 엑셀 검증과 병합은 LLM 판단이 아니라 pandas와 openpyxl 기반의 규칙 기반 로직으로 처리합니다.
- 엑셀 원본 파일은 보존하고, 결과 파일은 별도로 생성합니다.

6. Report Agent
- 검증 결과, 오류 내역, 병합 결과를 요약하여 최종 결과를 생성합니다.
- 최종 응답은 요약, 상세 결과, 확인 필요 사항, 생성 파일 정보 순서로 제공합니다.

항상 다음 원칙을 지켜야 합니다.

1. Supervisor Agent가 전체 업무 흐름과 상태를 관리합니다.
2. 각 전문 Agent는 자신의 역할에 맞는 작업만 수행합니다.
3. 1차 PoC에서는 mock 메일과 샘플 엑셀 파일을 사용합니다.
4. 사내 메일 직접 연동은 보안 정책과 접근 권한 문제로 1차 PoC 범위에서 제외합니다.
5. RAG는 검증 기준 확인이 필요한 경우에만 제한적으로 사용합니다.
6. 엑셀 검증과 병합은 LLM 판단이 아니라 규칙 기반 로직으로 처리합니다.
7. 오류 데이터는 자동 수정하지 않고 오류 보고서에 기록합니다.
8. 엑셀 원본 파일은 보존하고, 병합 파일과 오류 보고서는 별도로 생성합니다.
9. 주요 Agent 실행, 오류 발생 지점, 검증 결과는 로그 또는 Langfuse Trace로 기록합니다.
10. 회사 내부 정보, 개인정보, 업무상 민감한 정보는 외부로 노출하지 않습니다.
```

| 결과 요약       | Report Agent가 최종 검증 결과와 병합 결과를 요약                         |

#### Step 3 Execution & Response

Supervisor Agent는 각 전문 Agent의 실행 결과를 종합하여 최종 응답을 생성한다.

최종 응답은 다음 구조를 따른다.

```text
1. 요약
- 전체 처리 결과를 한눈에 보여준다.

2. 메일 분석 결과
- 요청 목적, 제출 기한, 작성 항목, 주의사항을 표시한다.

3. 엑셀 검증 결과
- 총 파일 수, 총 행 수, 정상 행 수, 오류 행 수, 오류 유형을 표시한다.

4. 생성 파일
- 병합 파일, 오류 보고서 경로를 표시한다.

5. 확인 필요 사항
- 재제출이 필요한 파일 또는 사용자가 확인해야 할 오류를 표시한다.

6. 실행 추적 정보
- 현재 실행 Agent, LangGraph 현재 Node, 로그 경로 또는 Langfuse Trace ID를 제공한다.
```

#### LangGraph Node / Edge 흐름

1차 PoC에서는 전체 Agent 흐름 중 엑셀 취합 핵심 기능에 필요한 Node만 우선 구현한다.

```text
START
  ↓
Supervisor Agent
  ↓
Email Input Node
  - mock email input
  ↓
Requirement Analysis Agent
  - request extraction
  - required fields extraction
  ↓
Supervisor Agent
  - validation rule planning
  - RAG required check
  ↓
조건 분기
  ├─ RAG 불필요 → Excel File Input Node
  └─ RAG 필요 → RAG Reference Agent → Excel File Input Node
  ↓
Excel Validation Agent
  - required field validation
  - date format validation
  - duplicate validation
  - code value validation
  ↓
Excel Merge Node
  - merge valid rows only
  ↓
Error Report Node
  - generate error report
  ↓
Report Agent
  - generate result summary
  ↓
Supervisor Agent
  - final response
  - trace logging
  ↓
END
```

### 지식 베이스 및 메모리 전략 Context & Memory

#### 4.1 RAG 검색 증강 생성 전략

본 프로젝트에서는 RAG를 전체 기능에 무조건 적용하지 않고, 회사 내부 기준이나 과거 사례를 참고해야 하는 기능에 제한적으로 적용한다.

1차 PoC에서는 RAG를 필수 기능으로 두지 않는다.\
단순히 메일에서 작성 항목을 추출하고 엑셀 파일을 검증하는 경우에는 RAG를 호출하지 않는다.

다만 다음과 같은 경우에는 RAG Reference Agent를 선택적으로 호출한다.

```text
작성 항목의 의미가 불명확한 경우
컬럼별 입력 기준이 필요한 경우
긴급도, 상태값 등 코드값 기준이 필요한 경우
날짜 형식 기준이 필요한 경우
검증 규칙 샘플 문서가 필요한 경우
```

반면 엑셀 병합, 필수값 검증, 날짜 형식 검증, 중복 데이터 확인은 LLM이나 RAG가 아니라 Excel Validation Agent가 pandas와 openpyxl 기반의 규칙 기반 로직으로 처리한다.

#### RAG 적용 여부

| 기능          | RAG 적용 여부     | 이유                                        |
| ----------- | ------------- | ----------------------------------------- |
| 취합 요청 메일 분석 | 미적용 또는 선택적 적용 | 기본적으로 메일 본문만으로 분석하고, 작성 항목 의미가 불명확할 때만 사용 |
| 검증 규칙 생성    | 부분 적용         | 코드값, 날짜 형식, 컬럼 기준이 내부 문서에 있을 때만 사용        |
| 엑셀 데이터 검증   | 미적용           | 필수값, 중복, 날짜 형식은 규칙 기반 검증이 적합              |
| 엑셀 병합       | 미적용           | pandas/openpyxl 기반 처리                     |
| 오류 보고서 생성   | 미적용           | 검증 결과를 구조화하여 보고서 생성                       |

### 최종 요약

Smart Collect Multi-Agent System은 엑셀 취합 업무에서 반복적으로 발생하는 작성 기준 불일치, 필수값 누락, 중복 데이터, 형식 오류를 줄이기 위한 멀티 에이전트 기반 업무 자동화 시스템이다.

이번 1차 PoC의 핵심 시나리오는 **취합 요청 메일 분석 → 제출 엑셀 파일 검증 → 정상 데이터 병합 → 오류 보고서 생성**이다.

전체 시스템은 Gmail MCP, RAG, 제출 추적, 리마인드, 최종 보고까지 확장 가능하도록 설계하지만, 1차 PoC에서는 실제 구현 범위를 엑셀 취합 핵심 기능으로 압축한다.

Supervisor Agent는 전체 흐름과 상태를 관리하고, Requirement Analysis Agent는 취합 요청 메일에서 작성 항목과 제출 기준을 추출한다.

Excel Validation Agent는 pandas/openpyxl 기반 규칙 로직으로 필수값 누락, 날짜 형식 오류, 중복 데이터, 코드값 오류를 검증한다.

검증을 통과한 데이터는 병합 파일로 생성하고, 오류 데이터는 별도의 오류 보고서로 생성한다.

RAG는 모든 단계에 적용하지 않고, 컬럼 기준이나 코드값 기준이 불명확한 경우에만 선택적으로 사용한다.

사내 메일은 보안 정책과 접근 권한 문제로 1차 PoC에서 직접 연동하지 않고, mock 메일 또는 Gmail MCP Adapter 구조로 대체한다.

이를 통해 1차 PoC에서는 구현 범위를 명확히 고정하면서도, 프로젝트의 핵심 가치인 **엑셀 취합 업무 자동화**를 실제로 검증하는 것을 목표로 한다.
