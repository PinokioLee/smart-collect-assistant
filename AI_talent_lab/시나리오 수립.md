### 시나리오 1 : AI 기반 취합 요청 메일 분석 및 작성 가이드 생성

* ID : SC-001

* 상황 :\
  취합 담당자가 요청자로부터 엑셀 자료 취합 요청 메일을 받았을 때 사용한다.\
  메일 안에는 작성 목적, 작성 항목, 제출 기한, 첨부 양식 등이 포함되어 있지만 내용이 길거나 복잡하여 작성자에게 그대로 전달하기 어렵다.

* 목표 :\
  AI가 취합 요청 메일 내용을 분석하여 작성 대상자들이 쉽게 이해할 수 있는 작성 가이드를 자동 생성한다.

* 사전 조건 :

  * 취합 요청 메일이 도착해 있어야 한다.

  * 요청 메일에 작성 기준, 마감일, 첨부 파일 정보가 포함되어 있어야 한다.

  * 시스템이 메일 본문 또는 첨부 파일 내용을 읽을 수 있어야 한다.

* 상세 흐름

| 단계 | 사용자 행동                       | 시스템 동작                                      | 비고                        |
| -- | ---------------------------- | ------------------------------------------- | ------------------------- |
| 1  | 취합 담당자가 취합 요청 메일을 시스템에 등록한다. | 시스템이 메일 제목, 본문, 첨부 파일 정보를 읽는다.              | Outlook 메일 연동 또는 수동 입력 가능 |
| 2  | 사용자가 “요청 내용 분석” 버튼을 클릭한다.    | AI가 요청 목적, 작성 항목, 제출 기한, 작성 대상, 주의사항을 추출한다. | LLM 기반 메일 분석              |
| 3  | 사용자가 분석 결과를 확인한다.            | 시스템이 작성자용 쉬운 작성 가이드를 자동 생성한다.               | 초등학생도 이해할 정도로 쉽게 작성       |
| 4  | 사용자가 일부 내용을 수정한다.            | 시스템이 수정 내용을 반영하여 최종 안내문을 생성한다.              | 담당자 최종 검토 단계              |
| 5  | 사용자가 작성 요청 메일 초안을 확인한다.      | 시스템이 작성자에게 보낼 메일 제목과 본문을 자동 완성한다.           | 발송 전 승인 필요                |

* 입력 예시

```text
메일 제목: 2026년 6월 부서별 시스템 개선 요청사항 취합 요청

메일 본문:
각 부서별로 2026년 6월 시스템 개선 요청사항을 취합하고자 합니다.
첨부된 양식에 맞춰 부서명, 담당자, 요청 시스템, 개선 요청 내용, 긴급도, 요청 사유를 작성해주시기 바랍니다.
제출 기한은 2026년 6월 12일 17시까지입니다.
작성 완료 후 본 메일에 회신 부탁드립니다.

첨부 파일:
시스템_개선요청_취합양식.xlsx
```

* 기대 출력

```text
[AI 분석 결과]

1. 요청 목적
- 2026년 6월 부서별 시스템 개선 요청사항을 취합하기 위함

2. 작성 대상
- 각 부서 담당자

3. 작성 항목
- 부서명
- 담당자
- 요청 시스템
- 개선 요청 내용
- 긴급도
- 요청 사유

4. 제출 기한
- 2026년 6월 12일 17시까지

5. 제출 방법
- 작성 완료 후 메일 회신

6. 작성자용 안내문
안녕하세요.
2026년 6월 시스템 개선 요청사항을 취합하고 있습니다.
첨부된 엑셀 파일을 열고 아래 항목을 작성해주세요.

- 부서명: 본인 부서명을 입력해주세요.
- 담당자: 작성자 이름을 입력해주세요.
- 요청 시스템: 개선이 필요한 시스템명을 입력해주세요.
- 개선 요청 내용: 어떤 점을 개선해야 하는지 구체적으로 작성해주세요.
- 긴급도: 상 / 중 / 하 중 하나를 선택해주세요.
- 요청 사유: 왜 개선이 필요한지 간단히 작성해주세요.

제출 기한은 2026년 6월 12일 17시까지입니다.
작성 완료 후 본 메일에 회신 부탁드립니다.
```

* 성공 기준

  * 기준 1: AI가 메일 본문에서 요청 목적, 작성 항목, 제출 기한을 정확히 추출한다.

  * 기준 2: 작성자가 이해하기 쉬운 작성 가이드가 자동 생성된다.

  * 기준 3: 취합 담당자가 메일 내용을 다시 정리하는 시간이 기존 대비 70% 이상 감소한다.

### 시나리오 2 : 작성자 대상 취합 요청 메일 및 알림 발송

* ID : SC-002

* 상황 :\
  취합 담당자가 작성 가이드를 완성한 후, 여러 작성자에게 취합 요청 메일을 보내야 하는 상황이다.\
  기존에는 작성자별로 메일을 작성하거나, 내용을 복사해서 반복 발송해야 했다.

* 목표 :\
  작성 대상자에게 취합 요청 메일을 자동 생성하고, 필요 시 SMS 또는 사내 알림을 함께 발송한다.

* 사전 조건 :

  * 작성자 목록이 등록되어 있어야 한다.

  * 작성 가이드가 생성되어 있어야 한다.

  * 제출 기한이 설정되어 있어야 한다.

  * 메일 발송 기능 또는 알림 발송 기능이 사용 가능해야 한다.

* 상세 흐름

| 단계 | 사용자 행동                 | 시스템 동작                                | 비고               |
| -- | ---------------------- | ------------------------------------- | ---------------- |
| 1  | 취합 담당자가 작성자 목록을 등록한다.  | 시스템이 이름, 부서, 이메일, 연락처 정보를 저장한다.       | 엑셀 업로드 또는 직접 입력  |
| 2  | 사용자가 발송할 작성 가이드를 선택한다. | 시스템이 작성자용 메일 제목과 본문을 자동 생성한다.         | AI 생성 메일         |
| 3  | 사용자가 제출 기한을 설정한다.      | 시스템이 마감일 기준으로 알림 일정을 자동 설정한다.         | 마감 1일 전, 3시간 전 등 |
| 4  | 사용자가 발송 전 내용을 확인한다.    | 시스템이 수신자별 메일 미리보기를 제공한다.              | 오발송 방지           |
| 5  | 사용자가 발송 버튼을 클릭한다.      | 시스템이 작성자들에게 메일을 발송하고 SMS 또는 알림을 전송한다. | 발송 이력 저장         |

* 입력 예시

```text
작성자 목록:
1. 김철수 / 생산1팀 / kimcs@company.com / 010-1111-1111
2. 박영희 / 품질팀 / parkyh@company.com / 010-2222-2222
3. 이민수 / 물류팀 / leems@company.com / 010-3333-3333

제출 기한:
2026년 6월 12일 17시

발송 내용:
2026년 6월 시스템 개선 요청사항 취합 안내
```

* 기대 출력

```text
[메일 제목]
2026년 6월 시스템 개선 요청사항 취합 요청드립니다.

[메일 본문]
안녕하세요.

2026년 6월 시스템 개선 요청사항을 취합하고 있습니다.
첨부된 엑셀 양식에 맞춰 아래 항목을 작성해주세요.

1. 부서명
2. 담당자
3. 요청 시스템
4. 개선 요청 내용
5. 긴급도
6. 요청 사유

제출 기한은 2026년 6월 12일 17시까지입니다.
작성 완료 후 본 메일에 회신 부탁드립니다.

감사합니다.

[SMS 알림]
[취합 요청]
2026년 6월 시스템 개선 요청사항 작성을 요청드립니다.
제출 기한: 6월 12일 17시
자세한 내용은 메일을 확인해주세요.
```

* 성공 기준

  * 기준 1: 작성 대상자 전체에게 동일한 기준의 요청 메일이 발송된다.

  * 기준 2: 메일 발송과 함께 SMS 또는 알림이 정상 발송된다.

  * 기준 3: 취합 담당자의 반복 발송 업무 시간이 기존 대비 50% 이상 감소한다.

### 시나리오 3 : 제출 현황 자동 추적 및 미제출자 리마인드

* ID : SC-003

* 상황 :\
  작성자들에게 취합 요청을 보낸 후, 누가 제출했고 누가 제출하지 않았는지 확인해야 하는 상황이다.\
  기존에는 취합 담당자가 메일함을 직접 확인하면서 제출 여부를 수동으로 체크해야 했다.

* 목표 :\
  시스템이 제출 현황을 자동으로 추적하고, 마감 전까지 제출하지 않은 사람에게 자동으로 리마인드 알림을 보낸다.

* 사전 조건 :

  * 작성자 목록이 등록되어 있어야 한다.

  * 취합 요청 메일이 발송되어 있어야 한다.

  * 제출 기한이 설정되어 있어야 한다.

  * 회신 메일 또는 제출 파일을 확인할 수 있어야 한다.

* 상세 흐름

| 단계 | 사용자 행동                     | 시스템 동작                             | 비고               |
| -- | -------------------------- | ---------------------------------- | ---------------- |
| 1  | 취합 담당자가 제출 현황 화면을 확인한다.    | 시스템이 작성자별 제출 여부를 표시한다.             | 제출 / 미제출 / 오류 상태 |
| 2  | 작성자가 엑셀 파일을 작성하여 메일로 회신한다. | 시스템이 회신 메일과 첨부 파일을 감지한다.           | 파일명, 제출 시간 기록    |
| 3  | 시스템이 제출 파일을 확인한다.          | 작성자 상태를 제출 완료로 변경한다.               | 자동 상태 업데이트       |
| 4  | 마감 시간이 가까워진다.              | 시스템이 미제출자 목록을 자동 추출한다.             | 마감 1일 전, 3시간 전   |
| 5  | 취합 담당자가 리마인드 발송을 승인한다.     | 시스템이 미제출자에게 재촉 메일 또는 SMS를 발송한다.    | 자동 또는 승인 후 발송 가능 |
| 6  | 마감 이후 취합 담당자가 최종 상태를 확인한다. | 시스템이 제출 완료자, 미제출자, 오류 파일 목록을 요약한다. | 결과 보고서에 활용       |

* 입력 예시

```text
제출 대상자:
1. 김철수 - 제출 완료
2. 박영희 - 미제출
3. 이민수 - 제출 완료
4. 최지훈 - 미제출

마감일:
2026년 6월 12일 17시

현재 시간:
2026년 6월 12일 14시
```

* 기대 출력

```text
[제출 현황 요약]

전체 대상자: 4명
제출 완료: 2명
미제출: 2명
제출률: 50%

[미제출자 목록]
1. 박영희 / 품질팀 / parkyh@company.com
2. 최지훈 / 생산2팀 / choijh@company.com

[리마인드 메일]
안녕하세요.
시스템 개선 요청사항 취합 자료가 아직 제출되지 않아 안내드립니다.
제출 기한은 오늘 17시까지입니다.
첨부 양식을 작성하여 회신 부탁드립니다.

[SMS 알림]
[취합 재안내]
아직 자료가 제출되지 않았습니다.
오늘 17시까지 작성 후 메일 회신 부탁드립니다.
```

* 성공 기준

  * 기준 1: 작성자별 제출 여부가 자동으로 갱신된다.

  * 기준 2: 미제출자 목록이 정확하게 추출된다.

  * 기준 3: 마감 전 미제출자에게 리마인드 알림이 발송된다.

  * 기준 4: 제출 현황 확인 시간이 기존 대비 80% 이상 감소한다.

### 시나리오 4 : 엑셀 파일 자동 병합 및 데이터 검증

* ID : SC-004

* 상황 :\
  작성자들이 제출한 여러 개의 엑셀 파일을 하나의 최종 취합 파일로 만들어야 하는 상황이다.\
  기존에는 담당자가 파일을 하나씩 열어 데이터를 복사하고, 누락이나 중복이 있는지 직접 확인해야 했다.

* 목표 :\
  제출된 여러 엑셀 파일을 자동으로 하나의 파일로 병합하고, 필수값 누락, 형식 오류, 중복 데이터 등을 자동 검증한다.

* 사전 조건 :

  * 작성자들이 엑셀 파일을 제출해야 한다.

  * 제출된 엑셀 파일이 정해진 양식 또는 유사한 구조를 가져야 한다.

  * 병합 기준 컬럼이 정의되어 있어야 한다.

  * 검증 규칙이 설정되어 있어야 한다.

* 상세 흐름

| 단계 | 사용자 행동                      | 시스템 동작                          | 비고                              |
| -- | --------------------------- | ------------------------------- | ------------------------------- |
| 1  | 취합 담당자가 제출 완료된 파일 목록을 확인한다. | 시스템이 제출된 엑셀 파일을 목록화한다.          | 파일명, 제출자, 제출 시간 표시              |
| 2  | 사용자가 “엑셀 병합” 버튼을 클릭한다.      | 시스템이 각 엑셀 파일의 데이터를 읽는다.         | Python openpyxl 또는 pandas 활용 가능 |
| 3  | 시스템이 데이터를 하나로 합친다.          | 작성자, 부서, 파일명 정보를 함께 추가한다.       | 추적 가능성 확보                       |
| 4  | 시스템이 검증 규칙을 적용한다.           | 필수값 누락, 형식 오류, 중복 데이터를 확인한다.    | 오류 목록 생성                        |
| 5  | 사용자가 검증 결과를 확인한다.           | 시스템이 정상 데이터와 오류 데이터를 구분하여 보여준다. | 수정 요청 가능                        |
| 6  | 사용자가 최종 저장을 실행한다.           | 시스템이 최종 취합 파일과 오류 보고서를 생성한다.    | 최종 산출물 생성                       |

* 입력 예시

```text
제출 파일:
1. 생산1팀_시스템개선요청.xlsx
2. 품질팀_시스템개선요청.xlsx
3. 물류팀_시스템개선요청.xlsx

병합 기준 컬럼:
- 부서명
- 담당자
- 요청 시스템
- 개선 요청 내용
- 긴급도
- 요청 사유

검증 규칙:
- 부서명은 필수 입력
- 담당자는 필수 입력
- 요청 시스템은 필수 입력
- 긴급도는 상/중/하 중 하나만 입력 가능
- 동일 부서, 동일 요청 시스템, 동일 개선 요청 내용은 중복으로 판단
```

* 기대 출력

```text
[최종 취합 결과]

전체 제출 파일 수: 3개
전체 데이터 건수: 45건
정상 데이터: 42건
오류 데이터: 3건
중복 의심 데이터: 2건

[오류 상세]
1. 품질팀_시스템개선요청.xlsx
   - 5행: 긴급도 값 오류
   - 입력값: 매우 급함
   - 허용값: 상 / 중 / 하

2. 물류팀_시스템개선요청.xlsx
   - 8행: 요청 시스템 누락

3. 생산1팀_시스템개선요청.xlsx
   - 12행: 담당자 누락

[생성 파일]
1. 최종_시스템개선요청_취합본.xlsx
2. 오류검증보고서.xlsx
3. 제출현황보고서.xlsx
```

* 성공 기준

  * 기준 1: 여러 개의 엑셀 파일이 하나의 최종 파일로 정상 병합된다.

  * 기준 2: 필수값 누락, 형식 오류, 중복 데이터가 자동 검출된다.

  * 기준 3: 오류 상세 보고서가 자동 생성된다.

  * 기준 4: 최종 취합 작업 시간이 기존 대비 70% 이상 감소한다.

### 전체 시나리오 요약

본 시스템의 핵심 시나리오는 취합 요청 메일을 AI가 분석하고, 작성자에게 쉬운 작성 가이드를 제공하며, 제출 현황을 자동 추적하고, 최종적으로 여러 엑셀 파일을 하나의 파일로 병합 및 검증하는 흐름으로 구성된다.

이를 통해 취합 담당자는 반복적인 수작업을 줄이고, 작성자는 명확한 기준에 따라 자료를 작성할 수 있으며, 최종 취합 결과의 품질과 신뢰도를 높일 수 있다.
