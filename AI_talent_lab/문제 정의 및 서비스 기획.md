### 프로젝트 명

Smart Collect Assistant: Gmail 기반 자율형 취합 업무 멀티 에이전트


### 배경 및 문제 정의

* 현재 상황 (As-Is)

취합 담당자는 본 업무와 별개로 메일 확인, 요청 해석, 양식 준비, 안내 메일 작성, 회신 확인, 오류 반려, 미제출자 독촉, 엑셀 병합을 반복한다. 업무량보다 더 큰 문제는 메일이 올 때마다 사람이 다시 판단하고 다음 행동을 직접 수행해야 한다는 점이다.

1. 수신함을 열어 일반 메일, 취합 요청, 제출 회신, 질문, 스팸을 구분한다.
2. 취합 요청의 목적, 작성 항목, 마감일, 대상자를 읽고 다시 정리한다.
3. 기존 첨부 양식을 사용할지 새 양식을 만들지 판단한다.
4. 작성 가이드와 요청 메일을 작성하고 수신자를 확인해 발송한다.
5. 회신 파일을 취합 건과 연결하고 제출 여부를 갱신한다.
6. 파일별 필수값, 날짜, 숫자, 코드값, 중복을 확인한다.
7. 오류가 있으면 사유를 정리해 재작성 요청 메일을 보낸다.
8. 작성자의 질문에 원래 요청 내용을 찾아 답변한다.
9. 미제출자를 찾아 리마인드하고 정상 파일을 병합한다.
10. 기준 엑셀의 공통 값을 여러 파일에 반복 반영한다.

단순 매크로나 고정 순서 프로그램으로 일부 작업은 줄일 수 있지만, 메일마다 표현과 의도가 다르고 제출·질문·수정·연장 등 다음 행동이 달라진다. 반대로 모든 판단을 LLM에 맡기면 엑셀 검증의 재현성, 오발송 위험, 개인정보 보호 문제가 생긴다.

### 기회 요인: AI로 해결 가능한 이유

본 과제는 LLM을 단순 문장 생성기가 아니라 비정형 업무를 이해하고 다음 행동을 선택하는 판단 계층으로 사용한다. 다만 정확한 계산과 외부 행동은 결정적 코드와 정책으로 다시 검증한다.

| 문제 | 적용 기술 | 적용 이유 |
| --- | --- | --- |
| 다양한 표현의 메일 의미 파악 | Azure OpenAI Structured Output | 키워드만으로는 우회 표현과 문맥 의존 의도를 놓침 |
| 상황별 다음 행동 선택 | LangGraph Supervisor Agent | 제출·질문·연장·실패에 따라 실행 경로가 달라짐 |
| 양식이 없는 요청 처리 | Template Design Agent + LLM | 자연어 요구사항을 컬럼·형식·허용값 스키마로 변환 |
| 안내·반려·질문 답변 작성 | Communication/Q&A Agent + RAG | 원래 요청과 취합 Job의 사실에 근거한 자연어 작성 필요 |
| 엑셀 오류 검증 | pandas/openpyxl 규칙 엔진 | 동일 정확도에서 LLM보다 빠르고 결과가 재현 가능 |
| 안전한 자동 행동 | Policy Gate + Human-in-the-Loop | 신뢰도·수신자·도메인·첨부·위험 조건을 통과한 경우만 발송 |
| 실패 복구 | Observation Loop | Worker 오류를 관찰해 재시도 또는 사람 확인으로 경로 변경 |
| 감사와 개선 | SQLite Action Log + Langfuse | 메일 단위 판단, LLM 호출, Tool 실행, 지연, 오류를 추적 |


### 목표 사용자 (Target User)

* 여러 부서 또는 담당자에게 정기·비정기 자료를 요청하는 팀 기획 담당자
* 회신 파일의 제출 여부와 데이터 품질을 관리하는 취합 담당자
* 동일 기준값을 여러 엑셀 파일에 반영하고 최종본을 만드는 실무자

가장 우선하는 사용자는 취합이 본 업무가 아니지만 반복적으로 담당해야 하는 사내 실무자다.


### 프로젝트 목표

* 정성적 목표

  * 메일 도착부터 정상 제출 병합까지 사람의 반복 개입을 최소화한다.
  * LLM은 의미 판단과 자연어 생성에, 규칙 코드는 검증과 정책 집행에 사용한다.
  * 단순·저위험 업무는 자동 처리하고 불명확·고위험 업무만 사람에게 요청한다.
  * 각 에이전트의 판단과 행동을 추적 가능하게 남긴다.
  * 기존 엑셀 병합·공통 값 일괄 업데이트 기능을 메일 기반 취합 흐름과 연결한다.

### 핵심 기능 정의

1. **Gmail 수신함 자동 확인**: APScheduler가 화면에서 저장한 시간과 타임존에 맞춰 새 메일을 수집하며, 사용자가 `지금 실행`으로 즉시 실행할 수도 있다.
2. **메일 계층형 분류**: 일반 메일, 취합 업무, 스팸·위험 메일로 분류하고 취합 업무는 요청, 제출, 질문, 수정본, 연장 요청으로 세분화한다.
3. **Supervisor 자율 라우팅**: 현재 상태와 Worker 실행 결과를 보고 다음 Agent 또는 사람 확인을 선택한다.
4. **요구사항·양식 결정**: 메일에서 목적, 항목, 마감, 대상, 주의사항을 추출한다. 첨부 양식이 있으면 재사용하고 없으면 Excel 양식을 생성한다.
5. **업무 역할·수신자 관리**: 최초 요청자·참조자는 최종 회신 대상으로 별도 저장하고, 취합 작성자는 메일의 명시 대상 또는 조직도에서 결정한다. 승인 화면에서 수신자를 추가·삭제·교체할 수 있다.
6. **안내 메일 작성·발송**: 작성 가이드와 취합 Job ID를 포함한 메일을 생성한다. 첨부 양식 재사용 등 단순·저위험 건은 정책을 통과하면 자동발송하고, 신규 양식 생성·불명확·고위험 건은 승인 대기로 보낸다.
7. **질문 자동응답**: 작성자의 회신 질문을 Gmail thread 또는 Job ID로 연결하고, 원래 요청의 마감·작성 항목·검증 규칙에 근거해 답변한다. 기한·양식·예외 정책 변경은 사람 승인을 요청한다.
8. **제출 검증·재작성 요청**: 필수값, 날짜, 숫자, 코드값, 중복을 검사하고 오류 사실에 근거한 반려 메일을 작성한다.
9. **안전한 Self-Correction**: LLM이 날짜·코드값 교정을 제안하되 코드가 허용 범위를 검증하고, 오류 수가 실제 감소한 경우에만 별도 교정본을 채택한다.
10. **제출 현황·리마인드**: 정상 제출자를 완료 처리하고 마감 임박 미제출자에게 리마인드 초안 또는 정책을 통과한 메일을 보낸다.
11. **최종 통합 검증**: 작성자 전원 제출 후 모든 파일을 다시 묶어 파일 간 중복까지 검사하고 오류가 있으면 완료를 차단한다.
12. **자동 병합·최종 회신**: 최종 오류가 0건일 때만 Excel을 병합하고, Completion/Report Agent가 병합본을 첨부해 최초 요청자·참조자에게 원래 Gmail thread로 회신한다.
13. **기준 Excel 동기화**: 같은 키와 공통 컬럼을 기준으로 하나의 기준 Excel 값을 여러 대상 파일에 일괄 업데이트한다.
14. **관찰 가능성**: SQLite에 Job·Submission·Action Log를 저장하고 Langfuse에 LLM 입력·출력·토큰·지연·오류를 기록한다.
15. **웹 UI·수동 보완**: 자동 확인, 승인 큐, 처리 상태, Agent 이력과 함께 승인 메일의 제목·본문·수신자를 직접 수정할 수 있다.


### 기대 효과

* 사람이 모든 메일을 읽는 방식에서, 예외와 승인 대상만 확인하는 방식으로 전환
* 우회 표현이 포함된 메일의 의도 파악 정확도 향상
* 제출·질문·수정·연장에 따른 후속 작업 자동 연결
* 오류 반려와 미제출 리마인드의 누락 감소
* 결정적 검증으로 엑셀 품질과 재현성 확보
* 실패 시 무리하게 계속 실행하지 않고 재계획 또는 사람 확인으로 안전하게 종료
* 실행 기록을 근거로 운영 장애와 LLM 품질을 분석할 수 있음


### 범위 및 제약사항

* In Scope

  * Gmail API 기반 새 메일·첨부 수집과 동일 thread 답장
  * 화면 설정 기반 정기 실행과 수동 즉시 실행
  * 일반·취합·스팸 분류 및 5개 취합 의도 분류
  * 첨부 양식 재사용 또는 신규 Excel 양식 생성
  * 작성 가이드, 요청·반려·질문 답변·리마인드 메일 생성
  * 정책 기반 자동발송과 사람 승인 분기
  * 제출 검증, 제한적 Self-Correction, 재제출, 병합
  * 작성자 전원 제출 후 최종 교차 검증과 최초 요청자 완료 회신
  * 제출 현황과 미제출자 리마인드
  * 기준 Excel 기반 공통 값 일괄 업데이트
  * SQLite 실행 이력과 Langfuse LLM 관측
  * FastAPI + React 웹 서비스

* Out of Scope

  * SMS·메신저·사내 그룹웨어 실연동
  * 병합 셀, VBA 매크로, 암호화 파일 등 모든 Excel 형식의 완전 자동 처리
  * 조직의 승인·권한·보존 정책을 대신하는 기능
  * 근거가 없는 업무 내용의 임의 보완 또는 정책 변경 자동 승인
  * 운영 환경 전체 사용자 대상 무제한 자동발송

* 제약사항

  * 실제 운영 시 Gmail OAuth, 조직도, 허용 도메인, 접근 권한 설정이 필요하다.
  * 자동발송은 기본 비활성화이며 테스트 계정과 허용 도메인 검증 후 활성화해야 한다.
  * LLM 분류 결과는 모델·프롬프트·메일 분포 변화에 따라 재평가해야 한다.
  * 현재 100% 수치는 내부 통제 평가셋 결과이며 운영 무오류를 의미하지 않는다.
  * 사람 대비 시간 절감률은 동일 시나리오 수작업을 최소 3회 실측한 뒤 확정해야 한다.
